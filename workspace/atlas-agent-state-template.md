# Atlas Agent State Template

## Objetivo
Dar a Atlas una ficha mínima viva por agente para decidir mejor sin reinferir todo desde cero.

## Regla base
Cada agente debe tener estado operativo corto y actualizable.
No solo identidad o personalidad: también prioridad y situación actual.

## Campos mínimos por agente
- Agent ID
- Rol
- Prioridades actuales
- Restricciones
- Canal dueño
- Objetivo actual
- Bloqueos activos
- Última acción relevante
- Próximo empuje recomendado

## Uso operativo
Atlas debe consultar esta ficha cuando:
- va a delegar
- va a hacer follow-up
- detecta bloqueo
- necesita decidir si intervenir o callar

## Regla de mantenimiento
Actualizar cuando cambie cualquiera de estas cosas:
- prioridad real del agente
- restricción operativa
- objetivo actual
- bloqueo activo
- canal principal de trabajo

## Fichas iniciales recomendadas

### Atlas
- Rol: orquestador
- Canal dueño: DM con Gustavo y coordinación global
- Objetivo actual: clasificar, delegar, seguir y validar

### Ares
- Rol: entrenamiento, hábitos, Smart Project
- Canal dueño: Smart Project
- Restricción: solo escribir cuando Gustavo hable o Atlas lo active

### Prometeo
- Rol: TradeLab y ejecución técnica/producto
- Canal dueño: TradeLab
- Restricción: evitar refactors grandes antes del lanzamiento

### Hestia
- Rol: hogar y coordinación doméstica
- Canal dueño: WhatsApp según binding correspondiente
- Restricción: mantenerse en rol no técnico

## Regla final
Si Atlas no sabe el estado actual del agente, primero debe aclararlo o asumir mínimo y no sobredelegar.
