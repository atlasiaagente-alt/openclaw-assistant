# Atlas Delegation Template

## Objetivo
Estandarizar cómo Atlas delega trabajo a agentes para que cada handoff tenga:
- owner claro
- acción concreta
- canal definido
- criterio de cierre
- condición de escalamiento

## Regla base
Atlas no delega con instrucciones blandas tipo:
- revisa esto
- mira esto
- ayúdame con esto

Atlas delega con contrato operativo.

## Plantilla corta
Usar este formato cuando la tarea sea simple o urgente:

- Contexto: ¿qué está pasando?
- Tarea: ¿qué debes hacer exactamente?
- Canal: ¿dónde se ejecuta?
- Done: ¿cuándo cuenta como hecho?
- Escala si: ¿cuándo debes devolverlo a Atlas?

## Plantilla completa

### 1. Owner
- Agente responsable único

### 2. Contexto
- Qué pasó
- Qué señal activó la tarea
- Qué información mínima necesita el agente

### 3. Objetivo
- Qué resultado concreto se busca

### 4. Acción exacta
- Qué debe hacer el agente ahora
- Debe ser ejecutable, no interpretativa

### 5. Canal de ejecución
- Interno
- WhatsApp
- Notion
- TradeLab
- Smart Project
- Otro canal específico

### 6. Restricciones
- Qué no debe hacer
- Límites de tono, alcance o timing

### 7. Definición de done
Una tarea delegada cuenta como hecha solo si:
- se ejecutó la acción pedida
- si requería mensaje externo, hubo entrega real
- si no se pudo cerrar, quedó bloqueo explícito o siguiente paso claro

### 8. Evidencia esperada
- confirmación interna
- enlace
- cambio visible
- mensaje entregado
- estado actualizado

### 9. Escala si
- falta criterio que el agente no puede inferir
- hay conflicto con restricciones
- no existe canal claro
- la tarea se bloquea de verdad

## Ejemplo bueno
- Contexto: en Atlas OS apareció una tarea de entrenamiento para hoy con owner Ares.
- Tarea: decide el entrenamiento del día y envía una instrucción corta y concreta al grupo Smart Project.
- Canal: Smart Project.
- Done: el mensaje quedó enviado en el canal correcto.
- Escala si: falta información crítica sobre lesión, energía o restricción física.

## Ejemplo malo
- Ares, revisa esto y me cuentas.

## Regla operativa final
Toda delegación debe poder responder estas cinco preguntas:
- quién
- qué
- dónde
- cuándo cuenta como hecho
- cuándo debe escalar
